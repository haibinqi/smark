# 产品需求文档 PRD

## 一、产品背景与目标

用户希望拥有一个极简、无干扰的在线 Markdown 编辑与预览工具，用于日常写作、技术笔记和轻量文档输出。现有编辑器功能复杂、界面拥挤，降低了专注度。本产品聚焦于写作本身，只保留必要能力。

核心目标：

1. 打开即写，无学习成本
2. 视觉克制，长期书写不疲劳
3. Markdown 渲染结果与 GitHub 保持一致

## 二、目标用户

1. 开发者，用于技术笔记、README 草稿
2. 知识工作者，用于日常记录、方案草拟
3. 习惯 Markdown 写作，希望界面足够安静的用户

## 三、产品定位

这是一个在线 Markdown 编辑与预览页面，不涉及账户体系、不强调协作、不做复杂文档管理。单页应用，核心体验围绕输入与阅读。

## 四、整体布局与交互设计

### 4.1 页面布局

整体采用固定宽度、居中布局，宽度建议 720px 或 800px，适合正文阅读。

页面为三行结构，自上而下排列。

第一行：

- 左侧：极简菜单区
    - 新建文档
    - 清空内容
    - 导出 Markdown
- 右侧：当前日期显示，格式 YYYY-MM-DD，可选显示星期

第二行：

- 文档标题输入框
- 单行输入，支持 Markdown 行内语法
- 字号明显大于正文

第三行：

- Markdown 正文编辑区
- 编辑区与预览区可切换
- 默认显示编辑与预览上下排列，后续可扩展左右分栏

### 4.2 交互原则

1. 所有操作尽量不打断输入流
2. 无弹窗、无确认对话框
3. 操作反馈轻量化，例如 toast 或状态提示

## 五、核心功能需求

### 5.1 Markdown 编辑

1. 支持标准 Markdown 语法
    - 标题
    - 列表
    - 引用
    - 代码块
    - 表格
    - 图片链接
2. 支持常见快捷键
    - Ctrl 或 Cmd + B 加粗
    - Ctrl 或 Cmd + I 斜体
    - Tab 处理列表缩进
3. 输入体验流畅，编辑区为等宽字体

### 5.2 Markdown 预览

1. 实时预览
2. 渲染风格采用 GitHub Markdown Theme
3. 代码块支持基础高亮
4. 图片按 Markdown 标准方式渲染

### 5.3 文档操作

1. 新建文档
    - 清空标题与正文
2. 导出
    - 导出为 .md 文件
    - 文件名默认为标题
3. 自动保存
    - 使用浏览器本地存储
    - 页面刷新后内容可恢复

## 六、非功能性需求

### 6.1 性能

1. 页面首次加载时间小于 1 秒
2. 输入延迟不可感知
3. 预览渲染在 50ms 内完成

### 6.2 兼容性

1. 支持主流桌面浏览器
    - Chrome
    - Edge
    - Firefox
2. 移动端可访问，但不作为主要优化目标

### 6.3 可维护性

1. 模块结构清晰
2. Markdown 渲染与编辑解耦
3. 样式与逻辑分离

## 七、视觉与风格要求

1. 整体风格偏 Notion 与 GitHub 之间
2. 背景色浅色，推荐接近白色
3. 字体清晰，正文行高适中
4. 页面元素数量克制，避免视觉噪音

## 八、技术实现建议（非强制）

1. 前端框架：React 或同类轻量方案
2. Markdown 解析：remark + rehype 或 markdown-it
3. 主题样式：GitHub Markdown CSS
4. 数据存储：LocalStorage 或 IndexedDB

## 九、版本规划

V1.0

- 单页 Markdown 编辑与预览
- 固定布局
- 本地自动保存
- Markdown 导出

V1.1

- 编辑与预览布局切换
- 深色模式

V1.2

- 多文档本地管理
- 简单文档列表
